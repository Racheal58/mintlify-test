# Salsa as a Service API v2.0

<Info>
The v2.0 API adds GraphQL support, enhanced webhooks, and new AI forecasting endpoints.
</Info>

## Base URL

```
https://api.salsaasaservice.com/v2
```

<Note>
**New in v2.0**: GraphQL endpoint at `https://api.salsaasaservice.com/graphql`
</Note>


## Authentication

All API requests require authentication using your API key.

<Note>
**API Key**: Get your API key from the [Developer Settings](https://app.salsaasaservice.com/settings/api) in your dashboard.
</Note>


### Authentication Header

Include your API key in the Authorization header:

```http
Authorization: Bearer YOUR_API_KEY
```

## Inventory Management 

### Get All Ingredients

<Tabs>
  <Tab title="Get All Ingredients">
    Retrieve a list of all ingredients in your inventory.

    ```http
    GET /ingredients
    ```

    **Response:**

    ```json
    {
      "data": [
        {
          "id": "ing_123",
          "name": "Roma Tomatoes",
          "category": "vegetables",
          "current_stock": 25.5,
          "unit": "lbs",
          "supplier": "Fresh Farms Co.",
          "last_updated": "2024-01-15T10:30:00Z",
          "expiry_date": "2024-01-20T00:00:00Z",
          "quality_score": 8.5
        }
      ],
      "pagination": {
        "page": 1,
        "per_page": 50,
        "total": 1
      }
    }
    ```
  </Tab>
  <Tab title="Update Ingredient Stock">
    Update the stock level for a specific ingredient.

    ```http
    PUT /ingredients/{ingredient_id}/stock
    ```

    **Request Body:**

    ```json
    {
      "quantity": 30.0,
      "unit": "lbs",
      "notes": "New shipment received"
    }
    ```

    **Response:**

    ```json
    {
      "id": "ing_123",
      "name": "Roma Tomatoes",
      "current_stock": 30.0,
      "unit": "lbs",
      "updated_at": "2024-01-15T11:00:00Z"
    }
    ```
  </Tab>
  <Tab title="Add New Ingredient">
    Add a new ingredient to your inventory.

    ```http
    POST /ingredients
    ```

    **Request Body:**

    ```json
    {
      "name": "Jalape√±o Peppers",
      "category": "vegetables",
      "initial_stock": 5.0,
      "unit": "lbs",
      "supplier": "Spice Garden",
      "expiry_date": "2024-01-25T00:00:00Z"
    }
    ```
  </Tab>
</Tabs>

## Production Tracking 

<AccordionGroup> 
  <Accordion title="Start Production Batch">

    Begin tracking a new salsa production batch.

    ```http
    POST /production/batches
    ```

    **Request Body:**

    ```json
    {
      "recipe_id": "rec_456",
      "batch_number": "BATCH-2024-001",
      "planned_quantity": 5.0,
      "production_date": "2024-01-15T08:00:00Z",
      "assigned_chef": "chef_123"
    }
    ```

    **Response:**

    ```json
    {
      "id": "batch_789",
      "recipe_id": "rec_456",
      "batch_number": "BATCH-2024-001",
      "status": "in_progress",
      "started_at": "2024-01-15T08:00:00Z",
      "assigned_chef": "chef_123"
    }
    ```
  </Accordion>

  <Accordion title="Complete Production Batch">

    Mark a production batch as completed with quality metrics.

    ```http
    PUT /production/batches/{batch_id}/complete
    ```

    **Request Body:**

    ```json
    {
      "actual_quantity": 4.8,
      "quality_score": 9.2,
      "notes": "Excellent batch, perfect consistency",
      "completed_at": "2024-01-15T09:30:00Z"
    }
    ```
  </Accordion>

  <Accordion title="Get Production History">
    Retrieve production history with filtering options.

    ```http
    GET /production/batches?start_date=2024-01-01&end_date=2024-01-31&recipe_id=rec_456
    ```
  </Accordion>
</AccordionGroup>

## Error Handling

<Info>
  The API uses standard HTTP status codes and returns detailed error information.
</Info>


### Error Response Format

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing required fields",
    "details": {
      "missing_fields": ["name", "category"]
    }
  }
}
```

### Common Error Codes

<Tabs>
  <Tab title="Client Errors">
    - `400 Bad Request` - Invalid request data
    - `401 Unauthorized` - Missing or invalid API key
    - `403 Forbidden` - Insufficient permissions
    - `404 Not Found` - Resource not found
    - `422 Unprocessable Entity` - Validation errors
    - `429 Too Many Requests` - Rate limit exceeded
  </Tab>
  <Tab title="Server Errors">
    - `500 Internal Server Error` - Server error
    - `502 Bad Gateway` - Upstream service error
    - `503 Service Unavailable` - Service temporarily unavailable
  </Tab>
</Tabs>

## Webhooks

Configure webhooks to receive real-time notifications about important events.

<Tabs>
  <Tab title="Supported Events">
    <CardGroup cols={1}>
    <Card>
      `ingredient.low_stock` - When ingredient stock falls below threshold
    </Card>
    <Card>
      `production.batch_completed` - When a production batch is completed
    </Card>
    <Card>
      `quality.alert` - When quality scores fall below acceptable levels
    </Card>
    <Card>
      `inventory.expiry_warning` - When ingredients are approaching expiry
    </Card>
  </CardGroup>
  </Tab>
  <Tab title="Webhook Configuration">
    ```http
      POST /webhooks
      ```

      **Request Body:**

      ```json
      {
        "url": "https://your-app.com/webhooks/salsa",
        "events": ["ingredient.low_stock", "production.batch_completed"],
        "secret": "your_webhook_secret"
      }
    ```
  </Tab>
</Tabs>
